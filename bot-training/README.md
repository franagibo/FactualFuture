## bot-training: offline imitation-learning for the combat bot

This folder contains a minimal Python project to train a learned combat policy from the NDJSON dataset generated by the engine.

The flow is:

1. Use the engine script to collect data.
2. Create a Python venv and install dependencies.
3. Train a small MLP policy on the NDJSON.
4. Export weights to a JSON file matching the engine's `LearnedPolicyWeights`.
5. Load that JSON in Node/TypeScript and enable the learned bot in the simulator.

### 1. Collect data from the engine

From the repo root:

```bash
npm run collect:bot-data
```

This will create a file like:

- `data/imitation/imitation-gungirl-seed123456-N50.ndjson`

Each line is one decision:

```json
{
  "state": [ ... ],        // state feature vector
  "actions": [ [ ... ] ],  // per-action feature vectors
  "chosenIndex": 1         // index into actions[] chosen by the teacher bot
}
```

### 2. Create a Python virtualenv and install deps

From `bot-training/`:

```bash
python -m venv .venv
.\.venv\Scripts\activate   # Windows
# or: source .venv/bin/activate  # macOS/Linux

pip install -r requirements.txt
```

### 3. Train the policy

Still from `bot-training/`:

```bash
python train.py --data ../data/imitation/imitation-gungirl-seed123456-N50.ndjson
```

You can tweak options:

- `--epochs` (default `20`)
- `--batch-size` (default `64`)
- `--lr` learning rate (default `1e-3`)
- `--out` output PyTorch weights file (default `policy_best.pt`)

The script prints train/val accuracy and saves the best model to `policy_best.pt`.

### 4. Export weights for the engine

Once trained:

```bash
python export_weights.py \
  --data ../data/imitation/imitation-gungirl-seed123456-N50.ndjson \
  --weights policy_best.pt \
  --out learned-policy-gungirl.json
```

This produces a JSON file with:

```ts
interface LearnedPolicyWeights {
  W1: number[][];
  b1: number[];
  W2: number[][];
  b2: number[];
  W3: number[][];
  b3: number[];
}
```

wrapped as:

```json
{ "weights": { "W1": [...], "b1": [...], ... } }
```

### 5. Use the learned policy in the simulator

In your Node/TypeScript code (e.g. a small script or within your sim harness), you can load and apply the model:

```ts
import fs from 'fs';
import path from 'path';
import { runSimulation } from '../src/engine/simulator/runSimulator';
import { setLearnedPolicy } from '../src/engine/simulator/learnedPolicy';

// 1) Load the exported JSON
const policyJsonPath = path.join(__dirname, '..', 'bot-training', 'learned-policy-gungirl.json');
const policyConfig = JSON.parse(fs.readFileSync(policyJsonPath, 'utf8'));

setLearnedPolicy(policyConfig); // globally registers the model

// 2) Build SimulatorOptions similar to run-balance-sim.spec.ts
//    (load cards, enemies, mapConfig, etc.)

const { runs, winRate, avgFloorReached, avgHpAfterFirstCombat } = runSimulation(
  {
    characterId,
    charactersMap,
    mapConfig,
    cardsMap,
    enemyDefs,
    encountersMap,
    relicDefs,
    eventPool,
    rewardCardPool,
    useLearnedPolicyBot: true, // key flag
  },
  200,
  42
);
```

The engine will:

- Use `pickActionLearned` (backed by your exported MLP) when `useLearnedPolicyBot` is true and a model is configured.
- Fall back to the heuristic bot if the learned policy is missing or throws.
